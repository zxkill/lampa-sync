# Lampa Sync Player

Lampa Sync Player — self-hosted плагин и серверный backend для [Lampa](https://github.com/yumata/lampa). Проект добавляет собственный HTML5/HLS-плеер, синхронизацию прогресса между устройствами, выбор аудиодорожек и фоновую подготовку видео через TorrServer + FFmpeg.

Проект не содержит медиаконтента и не распространяет видео. Он работает только с теми ссылками и потоками, которые пользователь уже может открыть в своей Lampa/TorrServer-инсталляции.

## Возможности

- Запуск собственного HTML5/HLS-плеера поверх Lampa.
- Онлайн-просмотр TorrServer-потоков через серверный HLS/fMP4 или HLS/MPEG-TS в отдельных режимах.
- Поддержка `hls.js` для Chrome/Android/TV и нативного HLS там, где он доступен.
- Синхронизация прогресса просмотра между устройствами.
- Сохранение прогресса по стабильному stream-id: torrent hash/link, index и путь файла.
- Поддержка продолжения просмотра для онлайн-потоков и заранее подготовленных HLS-файлов.
- Синхронизация timeline Lampa через `file_view` / `file_view_<profile>`.
- Выбор аудиодорожки в плеере.
- Фоновая очередь подготовки видео.
- Страница очереди с прогрессом, статусами, retry/delete и кнопкой прямого просмотра готового файла.
- Добавление в очередь из плеера, со страницы очереди или из меню Lampa.
- Быстрое открытие уже подготовленных файлов через prepared-HLS.
- Настройки плагина внутри Lampa.
- Пресеты качества конвертации.
- Управление клавиатурой и пультом.

## Пресеты качества

Качество выбирается в настройках Lampa Sync внутри Lampa. По умолчанию используется сбалансированный режим.

Доступные пресеты:

| Пресет | Назначение |
| --- | --- |
| `lowcpu` | минимальная нагрузка на сервер, качество ниже |
| `fast` | быстрый режим с нормальным качеством |
| `balanced` | хорошее качество по умолчанию |
| `safe` | повышенное качество, выше нагрузка |
| `ultra` | CRF 18, до FullHD, максимальное качество среди стабильных режимов |
| `copy` | быстрый TS remux: копирование видео и аудио без перекодирования |

Режим `ultra` не увеличивает разрешение выше оригинала. Если исходник меньше FullHD, размер кадра сохраняется. Если исходник больше FullHD, видео ограничивается шириной до 1920px.

Режимы `lowcpu`, `fast`, `balanced`, `safe` и `ultra` используют перекодирование видео и аудио в браузерно-совместимый HLS/fMP4. Они лучше подходят для Chrome, Android, Android TV и устройств, где важна стабильная синхронизация звука и видео.

Режим `copy` предназначен для быстрого remux без потери качества: видео и аудио копируются как есть, а HLS собирается в MPEG-TS-сегменты. Он почти не нагружает CPU и сохраняет исходное качество, но совместимость зависит от кодеков исходного файла. Если аудио в формате AC3/EAC3/DTS или видео в кодеке, который не поддерживается конкретным браузером или устройством, воспроизведение может работать не везде. Для максимальной совместимости используйте `balanced`, `safe` или `ultra`.

## Субтитры

Интерфейс субтитров временно скрыт. Базовая заготовка для работы с субтитрами может присутствовать в коде, но в пользовательском интерфейсе выбор субтитров отключён до появления стабильной реализации.

## Требования

- PHP 8.1+.
- Расширения PHP:
  - `curl`
  - `pdo_sqlite`
  - `sqlite3`
- CLI PHP для worker-а и cron.
- `ffmpeg` и `ffprobe`.
- TorrServer, доступный с сервера по HTTP.
- Веб-сервер с HTTPS, если Lampa открывается по HTTPS.

## Установка

Скопируйте файлы проекта в каталог, доступный по HTTP:

```bash
/var/www/example.com/html/lampa-sync/
```

Создайте локальный конфиг:

```bash
cd /var/www/example.com/html/lampa-sync
cp config.example.php config.php
nano config.php
```

Минимальные настройки:

```php
const BASE_PUBLIC_URL = 'https://your-domain.example/lampa-sync';
const API_PUBLIC_URL = BASE_PUBLIC_URL . '/api.php';
const QUEUE_PUBLIC_URL = BASE_PUBLIC_URL . '/prepare_queue.html';

const TORRSERVER_LOCAL_BASE = 'http://127.0.0.1:8090';
const TORRSERVER_LOGIN = '';
const TORRSERVER_PASSWORD = '';

const FFMPEG_BIN = 'ffmpeg';
const FFPROBE_BIN = 'ffprobe';
```

Создайте рабочую директорию и выдайте права на запись пользователю веб-сервера и CLI worker-а:

```bash
mkdir -p data
chown -R www-data:www-data data
chmod -R 775 data
```

Для Apache/CentOS пользователь может называться `apache`:

```bash
chown -R apache:apache data
```

## Проверка

```text
https://your-domain.example/lampa-sync/api.php?debug=1
https://your-domain.example/lampa-sync/progress.php?user_id=1&profile=0&debug=1
https://your-domain.example/lampa-sync/prepare_queue.html
```

В диагностике API должны быть доступны `ffmpeg`, `ffprobe`, `curl` и SQLite.

## Подключение плагина в Lampa

Подключите JS-файл плагина:

```text
https://your-domain.example/lampa-sync/lampa-sync-plugin.js
```

Плагин определяет базовый URL по адресу собственного JS-файла. При нестандартной схеме подключения можно задать параметры вручную до загрузки плагина:

```js
window.LAMPA_SYNC_CONFIG = {
  baseUrl: 'https://your-domain.example/lampa-sync'
};
```

После подключения в настройках Lampa должен появиться раздел **Lampa Sync**. В нём можно выбрать пресет качества конвертации.

## Очередь подготовки

Страница очереди:

```text
https://your-domain.example/lampa-sync/prepare_queue.html
```

Добавить задачу можно несколькими способами:

1. Открыть видео в плеере и нажать кнопку подготовки.
2. Вставить ссылку на странице очереди.
3. В Lampa открыть меню действия у файла или серии и выбрать добавление в Lampa Sync.

Готовые задачи можно запустить прямо со страницы очереди кнопкой **Смотреть**. Плеер откроется в новой вкладке и будет использовать тот же механизм сохранения прогресса. При наличии служебных timeline-данных задача также обновляет `file_view`, чтобы при последующем открытии Lampa отображался актуальный прогресс.

Для prepared-задач используется выбранный на момент добавления пресет качества. Готовые HLS-файлы сохраняются в:

```bash
data/prepared/
```

Временные HLS-сессии онлайн-просмотра находятся в:

```bash
data/hls/
```

## Worker и watchdog

Worker можно запустить вручную:

```bash
php /var/www/example.com/html/lampa-sync/prepare_worker.php
```

Для стабильной работы очереди рекомендуется добавить cron:

```bash
* * * * * /usr/bin/php /var/www/example.com/html/lampa-sync/prepare_worker.php --watchdog >> /var/www/example.com/html/lampa-sync/data/prepare_cron.log 2>&1
```

Worker обрабатывает задачи по одной. Lock-файл защищает сервер от параллельного запуска нескольких тяжёлых `ffmpeg`-процессов.

## Синхронизация прогресса

Прогресс хранится в двух слоях:

1. Внутренний прогресс плеера через `api.php`.
2. Синхронизация localStorage Lampa через `progress.php`.

Для идентификации видео используется стабильный stream-id, сформированный из данных TorrServer-потока. Это помогает сохранять один и тот же прогресс для онлайн-просмотра и prepared-HLS, а также не смешивать разные фильмы или серии между собой.

`progress.php` синхронизирует ключи localStorage между устройствами, включая `file_view`, `file_view_<profile>`, историю просмотра и служебные данные Lampa.

## Основные endpoints

```text
api.php?debug=1
api.php?prepare_start=1
api.php?prepare_status=1&content_id=...
api.php?prepare_list=1
api.php?prepare_delete=1&content_id=...
api.php?prepare_retry=1&content_id=...
api.php?prepare_kick=1
api.php?prepare_reset_stalled=1
api.php?prepared_hls=1&key=...&file=index.m3u8

progress.php?user_id=...&profile=...&device_id=...
```

## Обновление

После обновления файлов рекомендуется очистить временные HLS-сессии:

```bash
rm -rf /var/www/example.com/html/lampa-sync/data/hls/*
```

Базы прогресса и очереди обычно удалять не нужно:

```text
data/progress.sqlite
data/prepare_queue.sqlite
```

После обновления JS-файлов нужно перезапустить Lampa или очистить кэш плагинов, иначе может продолжить работать старая версия скрипта.

## Безопасность перед публикацией

В репозиторий не должны попадать:

- `config.php`
- содержимое `data/`
- SQLite-базы
- HLS-сегменты
- логи
- реальные логины и пароли TorrServer
- приватные домены, IP-адреса и локальные пути сервера

Для этих файлов и директорий должен использоваться `.gitignore`.

## Ограничения

- Проект зависит от внутренних API Lampa, поэтому после обновлений Lampa отдельные хуки могут потребовать адаптации.
- Скорость онлайн-просмотра и фоновой подготовки зависит от TorrServer, доступности сидов, выбранного пресета качества и производительности сервера.
- Подготовленные HLS-файлы занимают место на диске.
- При сбое во время подготовки задача обычно пересобирается заново.
- Субтитры временно отключены в интерфейсе.
- Режим `copy` сохраняет исходное качество и работает быстро, но совместимость зависит от кодеков исходного файла и возможностей устройства.
- Некоторые онлайн-плагины Lampa могут отдавать ссылки в нестандартном формате.
